## How to Use

### 1. Start with Overview
Always begin by reading:
- `Project Overview.md` - High-level understanding
- `Architecture Overview/Architecture Overview.md` - System design

### 2. Navigate by Topic
Use the topic structure to find specific information:
- **API questions** ‚Üí `API Reference/` directory
- **Database questions** ‚Üí `Database Design/` directory  
- **Frontend questions** ‚Üí `Frontend Interface/` directory
- **Security questions** ‚Üí `Security Implementation/` directory

### 3. Cross-Reference with Source
The wiki documents the codebase but may become stale. Always verify critical details by:
- Checking actual source files referenced in wiki docs
- Running `grep_code` to find implementations
- Using `search_codebase` for semantic understanding

### 4. Update Awareness
The wiki is generated by Qoder's repowiki feature and reflects the codebase at generation time. Note:
- Check file timestamps to understand recency
- Look for discrepancies between wiki and actual code
- Report any outdated information to user

## Important Notes

- The wiki is **read-only** reference material
- For cross-agent documentation updates, use `docs/PRD.md` and `docs/TECH_STACK.md`
- The wiki supplements but doesn't replace reading actual source code
- Always verify wiki information against the live codebase when making changes

## Quick Reference

**Need to understand how something works?**
1. Check relevant wiki section
2. Verify with actual source code
3. Cross-reference with `docs/TECH_STACK.md` for technical decisions

**Need API details?**
1. Read `API Reference/` section
2. Check actual `api/*.js` files
3. Verify routes in `server.js`

**Need database structure?**
1. Read `Database Design/` section
2. Check `server.js` for DB initialization
3. Verify in `api/*.js` files for actual usage
