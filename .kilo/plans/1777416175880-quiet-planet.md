## Goal

Remove `electron` / `legacy` references from the project in the correct scope without making unintended architecture or build changes.

## Findings

- `AGENTS.md:7` still says: `Electron app is deprecated — retained in repo as logic reference only.`
- `docs/PRD.md` has multiple Electron/reference/legacy statements:
  - `PRD.md:13-15`
  - `PRD.md:89`
  - `PRD.md:105`
  - `PRD.md:109`
- `docs/TECH_STACK.md:9-14,41,51-52,59` still documents the Electron desktop/runtime stack as active architecture.
- `docs/DECISIONS.md:10` says Electron is retained as fallback/reference.
- `docs/PRODUCTION_READINESS.md:44` still mentions deprecated Electron as reference-only.
- Root `package.json` is still an Electron app package today:
  - `main: start.js`
  - Electron dependencies in `dependencies` and `devDependencies`
  - Electron packaging/start scripts in `scripts`

## Ambiguity To Resolve

The request can mean either of these:

1. Remove wording only
   - Update `AGENTS.md` and docs so they no longer mention `Electron` or `legacy`.
   - Keep the old Electron code and package setup in the repo untouched.

2. Remove the actual Electron app
   - Update docs/rules.
   - Remove Electron dependencies, scripts, and packaging config from root.
   - Potentially remove or archive Electron-era files such as `start.js`, `server.js`, `api/`, `assets/`, `forge.config.js`, `build.js`, installer-related files, and any runtime assumptions tied to them.
   - This is a much larger change with higher regression risk because some files may still be used as logic references.

## Recommended Implementation Paths

### Path A: Doc/rule cleanup only

Use this if the goal is to remove the instruction/reference language while keeping the old code present.

Steps:

1. Update `AGENTS.md` project notes to remove the Electron deprecation/reference line.
2. Update `docs/PRD.md` to remove `Reference only`, deprecation notes, and `legacy Electron app` wording.
3. Update `docs/TECH_STACK.md` so it reflects the current production target only, or clearly scopes the old root stack into an archival note if you still want historical traceability.
4. Update `docs/DECISIONS.md` and `docs/PRODUCTION_READINESS.md` to remove Electron/reference phrasing.
5. Re-scan repo docs for leftover `electron` / `legacy` wording and normalize language.

Validation:

- Search docs and `AGENTS.md` for `electron|Electron|legacy|Legacy`.
- Confirm the remaining hits are only legitimate domain terms like `Electronic sales journal`.

### Path B: Full Electron removal

Use this if the goal is to fully retire the Electron app from the repository.

Steps:

1. Perform all Path A doc/rule updates.
2. Audit root runtime/package ownership so the repo has a clear primary app entry after Electron removal.
3. Remove Electron-specific fields from root `package.json`:
   - `main: start.js`
   - Electron dependencies/devDependencies
   - Electron scripts (`electron`, `electron-build`, `package-win`, `start`, `package`, `make`, `publish`)
   - Electron-only `resolutions`/`overrides` entries where no longer needed
   - `credits: electronjs.org`
4. Audit and remove now-orphaned Electron files/configs if truly unused:
   - `start.js`, `server.js`, `forge.config.js`, `build.js`, `latest-yml.js`, `renderer.js`, `installers/`, possibly `api/`, `bridge/`, `assets/` depending on actual remaining web usage.
5. Update any docs or scripts that still assume root-level Electron commands.
6. Run targeted validation for the surviving app (`web-prototype/`) and ensure the root repo still has a coherent install/test/dev story.

Validation:

- Search repo for Electron references outside historical changelog/savepoint text.
- Verify root package scripts are still coherent.
- Run relevant web-prototype checks after implementation.

## Recommendation

Start with Path A unless you explicitly want the Electron code and packaging removed from the repository. The current repo still structurally depends on Electron at the root package level, so a full removal is not a small text change.

## Decision Needed

- Choose `Path A` to remove wording only.
- Choose `Path B` to remove the actual Electron app and legacy references from the repo structure.
