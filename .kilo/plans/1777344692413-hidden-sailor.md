# PPOS UI Responsiveness Improvement Plan

## Requirements Document

### Problem Statement
The PharmaPOS PH web prototype has significant responsive design issues when viewed on mobile (320px) and tablet (768px) viewports. The current design is desktop-first (1920px+) and degrades poorly at smaller sizes.

### Current State Analysis

| Viewport | Breakpoint | Status |
|----------|-----------|--------|
| Mobile (320×568) | ≤760px | **Broken** — sidebar overlays content, product cards overflow, cart panel unusable |
| Tablet (768×1024) | ≤920px | **Degraded** — toolbar collapses, product grid too narrow, cart panel stacked but cramped |
| Desktop (1920×900) | >1120px | **Good** — primary design target, layout works well |

### Key Files
- `web-prototype/src/app/globals.css` — All styles (2642 lines), 4 media queries at 1120px/920px/760px/print
- `web-prototype/src/components/pos-prototype.tsx` — Monolithic component (2255 lines) containing sidebar, all views, POS layout, cart panel
- `web-prototype/src/lib/use-pos-store.ts` — Zustand store (sidebar nav state, `navOpen`)

### Current CSS Architecture
- Pure CSS with custom properties (no Tailwind)
- Fixed sidebar: `15vw` width, `position: fixed`
- POS grid: `grid-template-columns: minmax(0, 1fr) 390px` (product area + cart)
- Product grid: `repeat(2, minmax(0, 1fr))` (2-column cards)
- Cart panel: `position: sticky; top: 18px`
- 3 responsive breakpoints: 1120px, 920px, 760px

---

## Identified Issues

### Critical (Mobile ≤760px)

1. **Sidebar doesn't auto-hide on load** — On mobile, the sidebar starts open (`navOpen` defaults to true in the store). Users must manually close it. No overlay backdrop, so content behind is partially visible and untappable.

2. **No mobile backdrop/overlay** — When sidebar is open on mobile, there's no dark overlay behind it. Tapping outside doesn't close it (no backdrop click handler).

3. **POS layout stacks but cart panel becomes static and scrolls away** — At 760px, `.pos-grid` becomes `grid-template-columns: 1fr` and `.cart-panel` becomes `position: static`. This means the cart scrolls out of view while browsing products, making it unusable for checkout.

4. **Product card overflow on small screens** — Cards contain: brand badge + product name + (OTC badge, SKU, stock, price) + chevron. At 320px, all these elements can't fit and overflow or wrap badly.

5. **Toolbar wraps poorly** — At 760px the toolbar becomes single-column, but still contains 5 controls (search, 2 selects, 2 buttons) that stack vertically, consuming excessive vertical space.

### Important (Tablet 760–1120px)

6. **Sidebar `15vw` can be too narrow** — At 920px viewport, sidebar is only ~138px. Navigation labels and user dropdown get cramped.

7. **Cart panel at 390px fixed width** — On screens 760–1120px, the cart takes over 50% of the workspace width after accounting for sidebar, leaving products too little room.

### Nice-to-Have

8. **Status indicators ("Storage denied", "Printer Offline")** take valuable topbar space on mobile — should collapse into icons or a dropdown.

9. **Navigation has 8 items** ("POS", "Products", "Customers", "RX Workspace", "Control Tower", "Settings", "Reports", "Sync Online") — no icon support, just text buttons.

10. **No bottom navigation or FAB on mobile** — Common mobile POS patterns use bottom nav or floating action buttons for quick access.

## Design Plan

### Mobile (<760px) — Phone Layout

**Sidebar:**
- Auto-close on initial load for mobile viewports
- Add a semi-transparent backdrop overlay (`rgba(0,0,0,0.5)`) that closes sidebar on tap
- Sidebar width: `min(260px, 85vw)` (slightly wider than current `min(220px, 80vw)`)

**POS Layout:**
- Convert cart panel to a bottom sheet/drawer that slides up when tapped
- Add a floating "Cart (0)" FAB button at bottom-right showing item count
- Product grid: switch to single-column on screens ≤480px
- Product cards: stack vertically (badge on top, content below) on ≤480px

**Toolbar:**
- Collapse to expandable search bar with filter icon toggle on ≤480px
- Keep 2-column on 481–760px

**Topbar:**
- Replace status indicators with compact icon-only pill badges
- Reduce heading size to `font-size: 20px`

### Tablet (760–1120px) — iPad Layout

**Sidebar:**
- Use `min(220px, 20vw)` — ensures readable minimum width
- Auto-collapse on load for 760–920px range

**POS Layout:**
- Cart panel width: reduce from 390px to `min(340px, 35%)` 
- Product grid stays 2-column but cards get compact layout (as already defined at 1120px breakpoint)

**Toolbar:**
- Maintain 2-column layout through 760–920px (currently drops to 1-column at 920px)

### Desktop (>1120px) — No Changes
Keep existing layout as-is. It works well at this size.

## Task Breakdown

### Task 1: Add sidebar backdrop overlay and close-on-outside-click
**Files:** `pos-prototype.tsx`, `globals.css`

- Add a `<div className="side-nav-backdrop" onClick={() => setNavOpen(false)} />` inside the `<main>` after the `<aside>`
- Style `.side-nav-backdrop`: `position: fixed; inset: 0; z-index: 9; background: rgba(0,0,0,0.5);` — hidden by default
- Show backdrop only when `.app-shell.nav-open` and viewport ≤760px via media query
- Add CSS transition for backdrop opacity

### Task 2: Auto-close sidebar on mobile load
**Files:** `pos-prototype.tsx`

- Use `useEffect` to detect `window.matchMedia("(max-width: 760px)")` on mount
- If viewport is ≤760px, call `setNavOpen(false)`
- Add a `resize` event listener that auto-closes sidebar when viewport shrinks to ≤760px
- Add a `change` listener on the media query list for efficiency

### Task 3: Mobile cart drawer (bottom sheet)
**Files:** `pos-prototype.tsx`, `globals.css`

- Add state: `const [cartOpen, setCartOpen] = useState(false)`
- On mobile (≤760px), render cart panel as a slide-up drawer from bottom instead of inline stacking
- Add CSS for `.cart-drawer`: `position: fixed; bottom: 0; left: 0; right: 0; max-height: 70vh; transform: translateY(100%); transition: transform 0.3s ease;`
- When `cartOpen` is true: `transform: translateY(0)`
- Add drag handle bar at top of drawer for visual affordance
- Add an overlay behind the drawer (same as sidebar backdrop pattern)

### Task 4: Floating cart FAB for mobile
**Files:** `pos-prototype.tsx`, `globals.css`

- Add a fixed "Cart" FAB button at bottom-right on mobile viewports
- Show item count badge: `Cart ({store.cart.length})`
- On tap, opens the cart drawer (sets `cartOpen(true)`)
- Hide the FAB when cart drawer is open
- Style: rounded, primary color, elevation shadow, `position: fixed; bottom: 20px; right: 20px;` — only visible ≤760px
- When cart has items, add a subtle pulse animation

### Task 5: Responsive product cards on small screens
**Files:** `globals.css`

- Add new breakpoint `@media (max-width: 480px)`:
  - `.product-card`: switch to vertical layout (badge on top, content below, no chevron)
  - `.product-grid`: switch to `grid-template-columns: 1fr`
  - `.product-name`: `font-size: 14px` (from current 14px, keep as-is)
  - `.product-meta-row`: `flex-direction: column; align-items: flex-start`
  - `.product-image`: `width: 40px; height: 40px; font-size: 12px`
- Adjust existing 760px breakpoint to use more compact card layout

### Task 6: Compact mobile toolbar
**Files:** `pos-prototype.tsx`, `globals.css`

- On ≤480px: render toolbar as a single search input with a filter icon button
- The filter icon expands/collapses the filter row (category + sort selects)
- Add state: `const [filtersOpen, setFiltersOpen] = useState(false)`
- Style filter row as a collapsible section below the search bar
- Move "Reprint Queue" and "Preview OR" buttons to a "..." overflow menu or icon buttons

### Task 7: Responsive topbar for mobile
**Files:** `globals.css`

- At ≤760px: reduce h1 to `font-size: 20px`, reduce subtitle to `font-size: 12px`
- At ≤480px: hide subtitle entirely, shrink nav-toggle button
- Status indicators: show as small icon-only pills (e.g., green dot + "Online", red dot + "Offline") instead of full text buttons
- Reduce padding on `.workspace` at ≤760px from 18px to 12px (already done, verify)

### Task 8: Improve tablet breakpoint (760–920px)
**Files:** `globals.css`

- Ensure sidebar uses `min(220px, 20vw)` width for 760–1120px range
- Cart panel: use `min(340px, 35%)` instead of fixed 390px for `.pos-grid` columns
- Adjust `.pos-grid` at this range: `grid-template-columns: minmax(0, 1fr) min(340px, 35%)`
- Keep toolbar as 2-column at 760–920px (currently drops to 1-column at 920px)

### Task 9: Touch-friendly interaction improvements
**Files:** `globals.css`

- Increase touch target size for all interactive elements on mobile
- Product cards: `min-height: 48px` at ≤760px
- Navigation buttons: `min-height: 44px`
- Cart drawer drag handle: `min-height: 44px` with large tap area
- Add `touch-action: manipulation` to buttons to remove 300ms tap delay

### Task 10: Test and verify across breakpoints
**Files:** none (testing only)

- Verify mobile (320px, 375px, 414px, 480px)
- Verify tablet (768px, 820px, 920px)
- Verify desktop (1120px, 1440px, 1920px)
- Verify sidebar open/close behavior at each breakpoint
- Verify cart drawer open/close on mobile
- Verify touch interactions
- Verify landscape orientation on mobile
